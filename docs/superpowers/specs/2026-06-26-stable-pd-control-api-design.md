# NewtonStablePDControlAPI — Design

Date: 2026-06-26

## Goal

Add USD authoring support for the `ControllerStablePD` actuator via a new
`NewtonStablePDControlAPI` codeless schema, so stable-PD (SPD) actuators can be
declared on `NewtonActuator` prims and read back through the Newton runtime's
`parse_usd`.

## Scope

This repository (`newton-usd-schemas`) is a codeless USD schema package. It ships
schema definitions, plugin registration, and authoring tests — it contains no
runtime code. The runtime `parse_usd` parser lives in the separate Newton runtime
repository and is **out of scope** here.

In this repo the "parsed back through `parse_usd`" requirement is validated the
same way every other control API is: USD authoring + read-back tests (apply the
API, set attributes, read the attributes back with their hard limits). The
runtime dispatch change is a follow-up in the Newton runtime repo.

Three files change, plus the changelog:

1. `newton_usd_schemas/generatedSchema.usda`
2. `newton_usd_schemas/plugInfo.json`
3. `tests/test_actuator.py`
4. `CHANGELOG.md`

## Schema definition

Add a new class after `NewtonPDControlAPI`, mirroring its structure. It is a
single-apply control-law API that auto-applies `NewtonActuatorControlBaseAPI`,
and is one of the mutually-exclusive control laws on a `NewtonActuator`.

```usda
class NewtonStablePDControlAPI "NewtonStablePDControlAPI" (
    apiSchemas = ["NewtonActuatorControlBaseAPI"]
    doc = """Stable PD (SPD) controller.

    A semi-implicit PD controller that computes effort using the next-step
    predicted state, giving improved stability at high gains compared to the
    explicit `NewtonPDControlAPI`.

    Control law: `effort = kp*(target_pos - (q + dt*v)) + kd*(target_vel - v)`"""
)
{
    float newton:kp = 0 (
        doc = """Proportional gain (in joint space). ... Range: [0, inf)"""
        limits = { dictionary hard = { float minimum = 0 } }
    )
    float newton:kd = 0 (
        doc = """Derivative gain (in joint space). ... Range: [0, inf)"""
        limits = { dictionary hard = { float minimum = 0 } }
    )
    int newton:numWorlds = 0 (
        doc = """Optional number of parallel simulation worlds for batched
        evaluation. 0 means infer from the simulation/batch. Range: [0, inf)"""
        limits = { dictionary hard = { int minimum = 0 } }
    )
}
```

### Attributes

| Attribute          | Type  | Default | Hard limit  | Notes |
|--------------------|-------|---------|-------------|-------|
| `newton:kp`        | float | `0`     | minimum `0` | Joint-space proportional gain; copied verbatim from `NewtonPDControlAPI`. |
| `newton:kd`        | float | `0`     | minimum `0` | Joint-space derivative gain; copied verbatim from `NewtonPDControlAPI`. |
| `newton:numWorlds` | int   | `0`     | minimum `0` | `0` = infer from simulation/batch; optional override for parallel-world batching. |

`constEffort`/feedforward are intentionally omitted to keep the schema minimal
(unlike `NewtonPDControlAPI`/`NewtonPIDControlAPI`).

## Plugin registration

Add a `NewtonPhysicsStablePDControlAPI` entry to `plugInfo.json` after the PD
entry, identical in shape:

```json
"NewtonPhysicsStablePDControlAPI": {
  "schemaIdentifier": "NewtonStablePDControlAPI",
  "alias": { "UsdSchemaBase": "NewtonStablePDControlAPI" },
  "autoGenerated": false,
  "bases": ["UsdAPISchemaBase"],
  "schemaKind": "singleApplyAPI",
  "apiSchemaCanOnlyApplyTo": ["NewtonActuator"]
}
```

## Tests

Add `TestNewtonStablePDControlAPI` to `tests/test_actuator.py`, mirroring
`TestNewtonPDControlAPI`:

- `test_api_registered` — plugin type + schema type name resolve to
  `NewtonPhysicsStablePDControlAPI` / `NewtonStablePDControlAPI`.
- `test_api_application` — applying the API auto-applies
  `NewtonActuatorControlBaseAPI` and creates `newton:kp`, `newton:kd`,
  `newton:numWorlds`.
- `test_kp`, `test_kd` — default `0`, set/get round-trip, hard minimum `0`,
  no maximum.
- `test_num_worlds` — default `0`, set/get round-trip, hard minimum `0`,
  no maximum (int variant).

These authoring + read-back tests are the in-repo stand-in for the runtime
`parse_usd` round-trip.

## Changelog

Add an unreleased section to `CHANGELOG.md` noting the new

## Out of scope

- `constEffort` / feedforward attributes.
